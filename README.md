# Warframe 仲裁解析器

本地解析 Warframe 仲裁模式日志文件，统计护盾无人机、敌人生成、每波/轮次详情，计算生息期望与效率评级。

**文件不会上传到任何服务器，全部在浏览器本地完成。**

---

## 功能

- **日志解析** — 拖入或选择游戏日志文件，自动识别仲裁任务
- **护盾无人机统计** — 总数、每波/轮次分布、每分钟产出
- **敌人生成计数** — 任务期间生成的敌人总数
- **生息期望计算** — 基于无人机掉率 + 轮次奖励，支持多种 Buff 加成
- **效率评级** — S / A+ / A / A- / F 五档，一眼看出刷子质量
- **时间模式切换** — 主机时间 / 客机时间 / 自定义时间，实时重算
- **波次/轮次详情** — 展开查看每个阶段的无人机数量与期望
- **多任务批量解析** — 一次解析多个任务，可设置显示最近 N 个

## Buff 加成

| Buff | 效果 |
|------|------|
| 掉落几率加成 | +100% |
| 繁荣 | +18% |
| 掉落数量加成 | +100% |
| 掉落祝福 | +25% |

## 支持的任务类型

防御、拦截、生存、挖掘、间谍、捕获、移动防御、破坏、歼灭、中断、救援、叛逃、刺杀

## 主题

内置 12 套主题，暗色系与亮色系各半：

| 暗色系 | 亮色系 |
|--------|--------|
| 暗色、米色、森林、樱花、深海、熔岩 | 亮色、薄荷、天晴、薰衣草、蜜桃 |

支持自定义背景图片（本地文件或 `backgrounds/` 文件夹），可分别调节背景透明度和内容透明度。

---

## 使用方法

1. 打开 `index.html`
2. 拖入或点击选择日志文件
3. 调整 Buff 选项和显示数量
4. 点击「开始解析」
5. 查看结果，可切换时间计算方式或展开波次详情

### 日志文件位置

- **Windows**: `%USERPROFILE%\AppData\Local\Warframe\EE.log`
- **Steam**: `%USERPROFILE%\AppData\Local\Low\Warframe\EE.log`

## 自定义背景

### 方式一：配置文件

1. 创建 `backgrounds/` 文件夹
2. 放入图片（建议 1920×1080，JPG 压缩到 500KB-1MB）
3. 编辑 `bg-config.js`，在 `BG_IMAGES` 数组中添加文件名

```js
const BG_IMAGES = [
    'my-bg-01.jpg',
    'my-bg-02.jpg',
];
```

### 方式二：页面内添加

点击设置菜单中的「+ 添加」按钮，直接选择本地图片。图片会保存在浏览器 IndexedDB 中，刷新后仍然保留。

## 技术栈

纯前端，零依赖：

- HTML5 + CSS3 + Vanilla JavaScript
- IndexedDB 存储用户上传的背景图
- localStorage 持久化主题、透明度等设置

## 评级标准

| 评级 | 1小时期望 |
|------|-----------|
| S | ≥ 800 |
| A+ | ≥ 700 |
| A | ≥ 600 |
| A- | ≥ 500 |
| F | < 500 |

> 期望 = (无人机数 × 6% × Buff乘数) + (轮次数 × 1.21)，再按任务时长换算到每小时。
